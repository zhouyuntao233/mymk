    后端开发流程

    首先在mybatis xml文件中编写Sql语句 增删改查等等

    然后在mapper中写对应的接口

    写完接口在service中调用接口

    再在serviceImpl中通过调用this.baseMapper.xxx实现接口

    之后在Facade中接收前端请求，转到Controller中，Controller调用Service

    Java8的使用

    记录一下常用的内容

    包装类的使用

    1. 包装类 Integer判断相等时 不能使用==而应该使用equals比较具体内容，**==**比较的两者内存地址是否相等，而包装类都是对内容的引用，在小于128的范围内，Integer是直接读取缓存的，大于128则是新建的对象，两个对象的地址是不一样的，源码如下。
    / * Cache to support the object identity semantics of autoboxing for values between
  * -128 and 127 (inclusive) as required by JLS.
  */
public static Integer valueOf(int i) {
    if (i >= IntegerCache.low && i <= IntegerCache.high)
        return IntegerCache.cache[i + (-IntegerCache.low)];
    return new Integer(i);
}

    数组的使用

    数组的复制

    int a[] = new int[]{18, 62, 68, 82, 65, 9};

int b[] = Arrays.copyOf(a, a.length);

int c[] = Arrays.copyOfRange(a,0,a.length);

// 好像数组的拷贝只适用于基本数据类型，对于其包装类(Wrapper Class) 是不适用的

    数组的排序

    Integer a[] = new Integer[]{18, 62, 68, 82, 65, 9};

// Java 8 之前的写法
Arrays.sort(a,new Comparator<Integer>(){
    @Override
    public int compare(Integer o1, Integer o2) {
        return o1.compareTo(o2);
    }
});
// Java 8 
Arrays.sort(a, ((o1, o2) -> o1.compareTo(o2)));
// 甚至可以替换成这样
Arrays.sort(a, (Integer::compareTo));

    总结一下就是 比较器Comparator前后顺序一样的就是顺序排列，不一样的就是逆序排列。

    Lambda

    Lambda是Java 8的一个语法糖，它可以对匿名内部类的写法进行简化。让我们不用关注是什么对象，而关注我们对数据进行了什么操作。

    基本格式

    参数为一个接口，且只有一个抽象方法。

    ( 参数列表 ) -> { 代码 }

// 新建线程 传统写法
new Thread(new Runnable() {
    @Override
    public void run() {
        System.out.println("new thread 1");
    }
}).start();

// Lambda写法
new Thread(()->{
    System.out.println("new thread 2");
}).start()

authorList.stream()
            .distinct() // 去重
            .filter(item->item.getAge()<18) // 过滤
            .forEach(item-> System.out.println(item)); // 打印
    map转换

    map主要做的是元素的转换

    authorList.stream()
            .map(item->item.getName()) // 将元素转化为什么
            .forEach(item-> System.out.println(item));
    sorted排序

    sorted主要是对元素的排序

    authorList.stream()
                .distinct()
                .sorted(((o1, o2) -> o1.getAge().compareTo(o2.getAge())))
                .forEach(item-> System.out.println(item.getAge()));
    limit限制

    limit主要是对元素个数进行限制

    authorList.stream()
                .distinct()
                .sorted(Comparator.comparing(Author::getAge))
                .limit(3) // 限制元素个数
                .forEach(item-> System.out.println(item));
    flatMap

    flatMap 转换元素的同时自动展平流

    authorList.stream()
                .flatMap(item->item.getBookList().stream()) //自动展平流
                .forEach(item-> System.out.println(item));

    MyBatis的缓存机制

    mybatis 默认开启了一级缓存，同一个SqlSession有效，SpringBoot 开启了事务，一级缓存才会生效。

    二级缓存需要手动开启，XML文件中加上<cache/>才能使用，关闭了SqlSession一级缓存会写入到二级缓存中。
